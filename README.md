Github版本命名规范
-----
版本提交分为如下两类：
> 创建（CREATE）
> 修改（UPDATE）

提交性质分为如下两类：
> 需求（STORY）
> BUG（BUG）

版本号将根据项目迭代次数周期性增长，比如0.1、0.2、1.1等等。
需求将在项目管理中列出。

综上，每次提交版本，必须填写Summary，格式为：

> [提交类型][提交性质][版本号]需求名称

比如：
        [CREATE][STORY][0.1]项目框架搭建
        表示新创建了一批项目框架有关的文件。

        [UPDATE][BUG][0.1]APP内存泄漏
        表示修改了关于APP内存泄漏相关的BUG。

另外，Description建议尽量填写详细。
